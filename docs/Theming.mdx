# Theming

All design tokens are CSS custom properties defined in `src/theme/`. Import `main.scss` once at your app root — it pulls in every token file and the base reset.

```scss
@use '@/theme/main.scss';
```

## ThemeProvider

Wrap your app in `ThemeProvider` to manage theme state. It reads from and persists to `localStorage` under the key `sp-theme`, and tracks the system preference automatically.

Dark mode is applied by setting `data-theme="dark"` on `document.documentElement`.

```tsx
import { ThemeProvider } from '@/theme/useTheme'

export function App() {
  return (
    <ThemeProvider>
      <YourApp />
    </ThemeProvider>
  )
}
```

Inside any component, use `useTheme()` to read or change the theme:

```tsx
import { useTheme } from '@/theme/useTheme'

function ThemeToggle() {
  const { theme, setTheme, resolved } = useTheme()

  return (
    <button onClick={() => setTheme(resolved === 'dark' ? 'light' : 'dark')}>
      {resolved === 'dark' ? 'Light mode' : 'Dark mode'}
    </button>
  )
}
```

## DirectionProvider

Wrap components (or your whole app) in `DirectionProvider` when you need RTL support. Base UI components read direction from this context instead of relying on the DOM `dir` attribute.

```tsx
import { DirectionProvider } from '@base-ui/react/direction-provider'

export function App({ locale }: { locale: string }) {
  const dir = locale === 'he' || locale === 'ar' ? 'rtl' : 'ltr'

  return (
    <DirectionProvider direction={dir}>
      <YourApp />
    </DirectionProvider>
  )
}
```

You only need `DirectionProvider` if you use Base UI components (Accordion, Select, Popover, etc.) in an RTL context. Plain HTML elements should still receive a `dir` attribute on a parent element — portalled content (modals, tooltips) needs both.
